# Granola setup

How to connect the app to your Granola meetings (Story 5). Granola's personal API
uses a **Bearer API key** (`grn_…`) — no OAuth. You generate it once in the Granola
app and paste it into `.env.local` (git-ignored, read server-side only). Simple like
Canvas: set it once, never touch it again.

What the app does with Granola: it lists your recent meeting **notes**, pulls each
**transcript**, and its **own AI** (the Anthropic API) reads them and generates your
**Work** action items. It deliberately ignores Granola's built-in action items. Once
you clear an item it's remembered as done and never regenerates.

## One-time setup

1. In the Granola desktop app, go to **Settings → Connectors → API keys**.
2. Click **Create new key**, choose the **Personal notes** scope, and **Generate**.
3. Copy the key (format `grn_…`) — treat it like a password.
4. Put it in `.env.local`:

   ```bash
   GRANOLA_API_KEY=grn_your_key_here
   ```

5. Restart `npm run dev`. Open **⚙︎ Settings → Granola** — it should read
   **Connected via API key**, and your Work list fills with AI-generated action items.

The API is Bearer-authenticated against `https://public-api.granola.ai/v1`
(`GET /notes`, `GET /notes/{id}?include=transcript`). Only notes that already have a
generated AI summary + transcript are returned. Personal keys can be revoked/updated
anytime in the same settings panel.

## AI extraction

Action items are generated by the **Anthropic API** (the same `ANTHROPIC_API_KEY`
the planner uses). Without a key, a **mock extractor** produces sample action items
so the flow still works.

## Demo mode (no key)

Set `GRANOLA_MOCK=1` to run against sample meetings/transcripts — useful for local
demos and screenshots without a real key:

```bash
GRANOLA_MOCK=1 CANVAS_MOCK=1 GOOGLE_MOCK=1 TOKEN_ENC_SECRET=dev npm run dev
```

## Notes

- The integration is **read-only** — the app never writes to Granola.
- Transcripts are sent only to the Anthropic API for extraction; they are never
  logged or committed. The persisted stores hold only action-item titles + source
  meeting names, and are git-ignored.
