# Granola setup

How to connect the app to your Granola meetings (Story 5). Like Google, this uses
**OAuth** — you connect once and the app silently refreshes access forever, so you
never reconnect. Secrets stay in `.env.local` (git-ignored) and are read
server-side only.

What the app does with Granola: it pulls your recent meeting **transcripts** and
its **own AI** (the Anthropic API) reads each one and generates your **Work** action
items. It deliberately ignores Granola's built-in action items. Once you clear an
item it's remembered as done and never regenerates.

## One-time setup

1. Create a Granola OAuth app (Granola developer settings) and note its
   **client ID** and **client secret**.
2. Register the redirect URI **exactly** as:
   `http://localhost:3000/api/granola/callback` (adjust host for deploy).
3. Fill `.env.local`:

   ```bash
   GRANOLA_CLIENT_ID=...
   GRANOLA_CLIENT_SECRET=...
   GRANOLA_REDIRECT_URI=http://localhost:3000/api/granola/callback
   # reuse the same TOKEN_ENC_SECRET you generated for Google
   TOKEN_ENC_SECRET=...
   ```

4. Start the app, open **⚙︎ Settings → Granola → Connect**, approve access once.
   The refresh token is stored **AES-256-GCM encrypted** in a git-ignored file and
   auto-refreshed from then on.

Granola's personal API issues short-lived (~1h) access tokens with a rotating
refresh token; the app handles that rotation for you. Endpoint URLs can be
overridden via `GRANOLA_AUTH_URL` / `GRANOLA_TOKEN_URL` / `GRANOLA_API_BASE` if
Granola's paths differ from the defaults.

## AI extraction

Action items are generated by the **Anthropic API** (the same `ANTHROPIC_API_KEY`
the planner uses). Without a key, a **mock extractor** produces sample action items
so the flow still works.

## Demo mode (no credentials)

Set `GRANOLA_MOCK=1` to run against sample meetings/transcripts — useful for local
demos and screenshots without connecting a real Granola account:

```bash
GRANOLA_MOCK=1 CANVAS_MOCK=1 GOOGLE_MOCK=1 TOKEN_ENC_SECRET=dev npm run dev
```

## Notes

- The integration is **read-only** — the app never writes to Granola.
- Transcripts are sent only to the Anthropic API for extraction; they are never
  logged or committed. The persisted stores hold only action-item titles + source
  meeting names, and are git-ignored.
